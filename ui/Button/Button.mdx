## Usage

```jsx
import { Button } from "#/ui/Button"
```

```jsx
<Button variant="primary">Primary</Button>
```

## Variants

The basic button comes with 5 variants

  <Button variant="outline">Outline</Button>
  <Button variant="ghost">Ghost</Button>

## Size

Each button has 6 different size options to choose between

    variant="primary"
    size="xs" // or sm, md, lg, xl, 2xl
  >
    Button
  </Button>
`}>
  <Button variant="primary" size="xs">xs</Button>
  <Button variant="primary" size="sm">sm</Button>
  <Button variant="primary" size="md">md</Button>
  <Button variant="primary" size="lg">lg</Button>
  <Button variant="primary" size="xl">xl</Button>
  <Button variant="primary" size="2xl">2xl</Button>

All of these different sizes are available for each variant

## Icons

You can place icons within a button by passing it as a child

<CodePreview code={`
    <Button variant="primary">
      <ArrowLeftIcon /> Previous
    </Button>
    <Button variant="primary">
      Next <ArrowRightIcon />
    </Button>
  `}>
  <Button variant="primary"><ArrowLeftIcon /> Previous</Button>
  <Button variant="primary">Next <ArrowRightIcon /></Button>
</CodePreview>

If you use an icon without text, make sure to include a hidden label for accessibility purposes

<CodePreview code={`
    <Button variant="destructive">
      <span className="sr-only">Delete</span>
      <Trash01Icon />
    </Button>
  `}>
  <Button variant="destructive">
    <span className="sr-only">Delete</span>
    <Trash01Icon />
  </Button>
</CodePreview>

## Interactivity

While buttons are often used within forms or as links, there are some instances where you may need to use them for something else.

### onClick

You can pass an `onClick` prop to a button to make it interactive, just like you can with a regular HTML button

<CodePreview code={`<Button variant="primary" onClick={() => alert('Button clicked!')
    Click me
  </Button>`}>
  <Button variant="primary" onClick={() => alert('Button clicked!')}>Click me</Button>
</CodePreview>

## Customization

### Classes

You can customize individual buttons by passing a `className` prop. If this clashes with the default styles, the `className` will take precedence.

<CodePreview code={`
    <Button
      variant="primary"
      className="bg-blue-500 hover:bg-blue-400 focus:bg-blue-400 border-blue-500"
    >
      Custom
    </Button>`}>
  <Button
    variant="destructive"
    className="bg-amber-500 hover:bg-amber-400 focus:bg-amber-400 border-amber-500"
  >
    {`Warning`}
  </Button>
</CodePreview>

Due to how `tailwind-merge` works, you won't need to worry about conflicting styles when using the `className` prop.

Make sure you override all the states to ensure a consistent experience, or you'll end up with something like this

<CodePreview>
  <Button variant="destructive" className="bg-amber-500">Warning</Button>
</CodePreview>
